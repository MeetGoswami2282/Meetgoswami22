Project Overview

This project is a mobile app redesign for a vendor management platform used to handle orders, bill reviews, commission invoices, wallet transactions, and account management.

Problem Statement

  • Overloaded screens with poor visual hierarchy.
  • Complex financial information (bills, commissions, wallet) was difficult to understand.
  • Important actions (Approve, Dispatch, Pay, Withdraw) were hard to notice.
  • Too many steps required to complete common tasks.
  • Inconsistent spacing, typography, and component styles.
  • Limited clarity in order tracking, bill status, and payment flow.

    • Project Goals

  • Simplify complex vendor workflows into clear, step-by-step experiences.
  • Improve readability and visual hierarchy across all screens.
  • Make key actions easy to find and faster to complete.
  • Enhance transparency for orders, bills, commissions, and wallet activity.

    • The Solution

  • Introduced clear card-based layouts for orders, bills, and invoices.
  • Improved status visibility using consistent color coding (Pending, Approved, Rejected, Paid).
  • Standardized typography, spacing, buttons, and icons for UI consistency.
  • Improved navigation clarity across Orders, Bills, Wallet, and Account sections.
